# WeApp 手工冒烟清单（P0）

> 目的：在不接入真实微信登录/支付前，验证 **微信小程序端** 的 P0 主页面可用、无阻塞错误、关键交互可走通（dev/demo 口径）。
> 适用：`scripts/start-dev.ps1 -EnableDemoAuth`（真实 API + Demo Auth）

## 1) 前置条件

- 已启动开发环境：`powershell -ExecutionPolicy Bypass -File scripts/start-dev.ps1 -EnableDemoAuth`
- API 正常：打开 `http://127.0.0.1:3200/health` 返回 `ok=true`
- WeChat DevTools 已打开 `apps/client` 工程，并可预览运行

建议记录：
- DevTools 版本（含 RC/Stable）
- 当前 API Base（应为 `http://127.0.0.1:3200`）

## 2) 冒烟用例（必测）

> 通过标准：页面可进入、关键数据可加载/提示合理、无明显报错弹窗/白屏；未登录态不应触发 401 风暴（应只提示登录引导）。

### A. 首页（Tab）
- [ ] 进入首页：首屏可见骨架/内容，未白屏
- [ ] Banner/快捷入口可点击跳转（至少 1 个）
- [ ] 列表区域可下拉刷新/加载更多（至少触发 1 次）

### B. 搜索
- [ ] 打开搜索页：默认推荐/热门词可见（或空态提示合理）
- [ ] 输入关键词搜索：结果列表正常渲染
- [ ] 切换筛选/排序：列表可刷新，条件变化生效（至少 1 次）

### C. 详情（从列表进入）
- [ ] 挂牌详情：信息区块可见，图片/附件入口显示正常
- [ ] 专利/需求/成果/书画任意 1 个详情页可进入且无报错
- [ ] 收藏/咨询入口可点击（未登录应提示登录；已登录应正常触发）

### D. 消息（Tab）/聊天
- [ ] 进入消息页：未登录应提示登录；已登录应展示会话列表或空态
- [ ] 进入聊天页：页面可进入，历史消息/空态正常

### E. 收藏
- [ ] 收藏列表：未登录提示登录；已登录可见列表/空态
- [ ] 在详情页点击收藏后，收藏列表可看到对应条目（或状态变更）

### F. 个人中心（Tab）
- [ ] 进入“我的”：未登录提示登录
- [ ] 登录（Demo）：勾选协议后可完成登录；登录后展示用户信息
- [ ] 进入资料编辑/认证入口：页面可进入、提示合理

### G. 发布（Tab）
- [ ] 进入发布页：入口可见
- [ ] 选择任意发布类型进入表单：页面可用、校验提示合理
- [ ] 试提交：未填写必填项应有明确校验；填写后可提交（若当前流程需要审核/草稿，提示合理即可）
