# 硬编码白名单（颜色/字号/间距）

> 目标：收敛“页面随手写样式”，让视觉能长期一致；允许极少数硬编码例外，但必须登记。

## 1. 原则

- 新增任何颜色/字号/间距/圆角/阴影硬编码前，先尝试落到 token（用户端 `apps/client/src/app.scss` / 后台 `apps/admin-web/src/styles.css` + AntD token）。
- 业务页面禁止写 `#fff/#000/rgba(...)` 等随手值（除白名单）。
- “无法避免的第三方组件必要值”允许硬编码，但需在本白名单登记并说明原因。

## 2. 允许的硬编码（白名单）

### 2.1 颜色

- `#ffffff`：仅用于图标/文字强制白色（例如 icon 的 `color="#fff"`），且无法通过 token 传入时。
- `rgba(255,255,255,0.92)` / `rgba(255,255,255,0.98)`：后台顶部栏/卡片的玻璃态容器背景（已收敛为 `--ipm-bg-container`、`--ipm-bg-elevated`）。
- 渐变/插画：Logo 渐变、品牌插画的内部色值（图片资源本身不受控）。

### 2.2 尺寸

- 触控热区：最小热区 `44px` 等效（用户端 `--tap-target=88rpx`，后台由 AntD 默认交互尺寸保障）。
- 少量像素级对齐：例如图标内边距、1px 边框（用户端用 `1rpx`，后台用 `1px`）。

## 3. 门禁建议（非强制）

- 开发自测前运行：`pnpm scan:banned-words`
- 若后续发现硬编码回潮：可追加脚本扫描 `#RRGGBB` 与 `fontSize/padding/margin` 等内联样式，作为 CI 门禁。
