# Ipmoney 管理后台运营使用说明书（简版 V1）

> 版本：`v1.0`  
> 生效日期：`2026-03-24`  
> 适用对象：运营、客服、财务、值班负责人  
> 配套审计：`docs/engineering/admin-backoffice-chain-audit-2026-03-24.md`

## 1. 使用目的
- 让运营同学在后台按统一步骤完成“审核、会话、工单、退款、发票、结算、权限”工作。
- 确保所有关键操作可追溯（有原因、有证据、有审计记录）。

## 2. 班前 5 分钟检查（必做）
1. 打开仪表盘，确认系统健康状态与待办数量。
2. 查看告警中心是否有未处理高优先级告警。
3. 查看平台会话中心是否存在“未分配会话”。
4. 抽查工单/退款是否有 SLA 逾期项。
5. 交接班时记录：未处理事项、风险订单号、下次更新时间。

## 3. 菜单职责速览
- `认证审核`：审核用户/机构资质，决定是否具备交易与咨询权限。
- `上架审核`：审核上架内容与发布质量（含批量导入任务）。
- `平台会话中心`：统一处理咨询/客服/争议会话，支持分配与连续沟通。
- `工单/争议`：处理跟单、退款争议、交易争议，沉淀证据与备注。
- `订单管理 / 订单详情`：处理里程碑确认（订金、合同、尾款、变更完成）。
- `退款管理`：审批退款、驳回退款、确认退款完成。
- `发票管理`：下发开票、上传/替换/删除发票附件。
- `放款/结算`：确认线下放款，上传放款凭证。
- `交易/推荐配置`：修改 banner、客服、字典、敏感词、热门搜索等配置。
- `账号权限`：员工开通、角色权限、账号授权（RBAC）。
- `审计日志`：追踪关键操作，支持复盘与合规检查。

## 4. 标准操作流程（SOP）

## 4.1 认证审核 SOP
**触发条件**：出现待审核认证。  
**操作步骤**
1. 进入 `认证审核`，打开详情查看材料与审核记录。
2. 核验主体信息、联系人、材料完整性。
3. 点击“通过”或“驳回”，必须填写明确原因（可复核）。
4. 核对状态变化是否成功，必要时刷新页面确认。  
**完成标准**：状态正确流转，审核理由可追溯。

## 4.2 会话处理 SOP
**触发条件**：未分配会话、用户催办、争议升级。  
**操作步骤**
1. 进入 `平台会话中心`，按“未分配/我的会话/渠道类型”筛选。
2. 先分配责任坐席（自己或指定人员），再回复消息。
3. 重要沟通必须留在会话中（避免口头无痕）。
4. 需要升级时同步创建/关联工单。  
**完成标准**：每条会话有责任人、有处理记录、有下一步动作。

## 4.3 工单与争议 SOP
**触发条件**：订单异常、退款争议、投诉。  
**操作步骤**
1. 在 `工单/争议` 新建或打开工单，补齐标题、类型、优先级、关联订单。
2. 设置分配人和 SLA 截止时间。
3. 上传证据附件，持续记录处理备注。
4. 按实际进展更新状态（`OPEN -> IN_PROGRESS -> CLOSED`）。  
**完成标准**：工单全程可回放，证据链完整。

## 4.4 订单履约 SOP
**触发条件**：到款确认、合同签署、变更完成。  
**操作步骤**
1. 在 `订单详情` 核对当前状态与金额。
2. 执行里程碑动作前进行二次确认（资金/合同/凭证）。
3. 提交时填写备注（到账凭证、合同编号、登记号等）。
4. 动作后刷新并确认状态已更新。  
**完成标准**：状态机连续、关键动作均有审计痕迹。

## 4.5 退款 SOP
**触发条件**：存在退款申请。  
**操作步骤**
1. 在 `退款管理` 输入订单号加载退款单。
2. 核验退款依据与争议材料。
3. 执行“通过/驳回/完成退款”并写明原因。
4. 更新工单与会话，通知相关方。  
**完成标准**：退款单状态闭环，资金动作与沟通记录一致。

## 4.6 发票与结算 SOP
**发票**
1. 在 `发票管理` 输入订单号，必要时先“下发开票”。
2. 上传/替换发票文件，补充票号与开票时间。
3. 删除发票时必须填写原因。  
**结算**
1. 在 `放款/结算` 输入订单号加载台账。
2. 上传放款凭证，填写流水号/备注。
3. 确认放款后复核状态与凭证关联。  
**完成标准**：附件可访问、凭证可审计、资金口径一致。

## 4.7 配置与权限 SOP
**配置变更**
1. 先确认变更目的与影响范围（活动页/搜索/合规词）。
2. 修改后执行双人复核（操作人 + 值班负责人）。
3. 记录变更时间与回滚方案。  
**权限变更**
1. 遵循最小权限原则开通员工账号。
2. 角色变更必须留原因，避免临时越权长期保留。
3. 每周抽查高危权限账号。

## 5. 异常升级与留痕模板（值班必填）
> 推荐每次重大异常至少记录以下字段（可直接贴到工单/群公告）：
- `Date/Time`：事件发生与发现时间
- `Reporter`：报告人
- `Severity`：严重级别（P1/P2/P3）
- `Impact`：影响范围（用户/订单/资金）
- `Actions Taken`：已采取动作
- `Current Status`：当前状态
- `Next Update`：下次同步时间

## 6. 当前版本能力边界（务必知晓）
- 当前项目口径为 dev/staging 演示联调，不是完整生产资金链路。
- 真实微信登录/支付/退款回调等生产关键项仍在上线收口清单中。
- 订单、退款、发票、结算目前以“订单号单查”为主，批量运营需配合工单与会话机制防漏。

## 7. 运营执行规范（简要）
- 每次高风险操作都要写“原因 + 证据线索”。
- 备注尽量结构化：`发生时间 + 核验依据 + 操作人`。
- 附件命名建议：`业务类型_订单号_日期_版本`。
- 不在会话外处理关键争议结论（避免不可追溯）。
