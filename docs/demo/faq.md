# 甲方 FAQ（演示口径）

## 1) 平台是否“托管资金”？风险怎么控？

- P0 采用“订金 + 尾款均在平台上支付并托管”的方式：订金表达意向，尾款在“合同签署确认后”才允许支付并托管。
- 放款条件固定为：**必须完成“权属变更完成确认”**后进入结算；P0 默认 **财务线下人工放款并回传凭证**，全链路留痕可对账。

## 2) 为什么合同要线下签？线上不行吗？

- P0 采用“线下签署合同（私法合同）+ 线上资金托管”的组合：降低电子签、身份核验、合同模板合规等成本，优先把交易闭环跑通。
- P1 可升级：接入电子签与在线签约流程（需法务与合规方案）。

## 3) 退款规则怎么定？自动退款窗口为什么是 30 分钟？

- P0 默认：订金支付后 **30 分钟内**，且客服未点击“开始跟单/线下处理”，买家可无理由申请退订金并系统秒退（原路退）。
- 进入跟单后：按证据与规则走人工审核（避免恶意退款/扯皮）；线下合同已签且无卖家违约时原则不可退。

## 4) 佣金谁承担？怎么算？

- 已确认：**佣金由卖家承担**，从卖家应收款中扣除。
- P0 默认：成交价 `5%`（`min=¥1,000`，`max=¥50,000`），可后台配置。

## 5) 平台如何核验“交易已成交”？

- 尾款在平台托管且需要后台“合同签署确认”才能支付，资金链路天然形成“成交事实”证据。
- 关键节点均需上传证据附件（合同关键页、变更完成证明、放款凭证等），并写入审计日志，便于仲裁/对账。

## 6) 是否允许多个卖家上架同一件专利？

- 允许同一 `application_no_norm` 多卖家上架。
- 风控方式：每条 listing 必须独立上传权属/授权材料并通过后台审核；支持举报与下架；审核/处置全留痕。

## 7) 专利数据来源是什么？是否对接官方检索？

- P0：专利信息以 **用户上传 + 后台录入/导入** 为准，不宣称“官方实时数据”。
- P1：如需自动回填/法律状态刷新/权威核验，再接 CNIPA/第三方数据源，走适配层隔离差异与合规边界。

## 8) 支付回调与对账如何保证可靠？

- 统一回调入口：必须验签、解密 `resource`，并做幂等（避免重复通知导致重复入账/重复退款）。
- 支付/退款/放款均要求：可追溯（台账 + 审计日志 + 证据附件），可对账（支付平台单号、放款流水号）。

## 9) 平台是否提供发票？开票范围是什么？

- P0 默认：订单完成后，平台可为该订单的**平台服务费/佣金**开具电子发票（电子普票）；系统内仅做“上传/下载”，不做线上申请流转。
- 平台不对专利转让/许可**标的价款**开票；如需标的价款发票，应由买卖双方线下另行约定并由卖家开具（或按合同约定处理）。
- 财务后台处理：线下收集抬头/税号等信息 → 线下开票 → 上传并绑定电子发票文件（PDF/图片）到已完成订单 → 卖家在平台内下载，过程留审计日志。
