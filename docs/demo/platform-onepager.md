# Ipmoney 专利交易平台（P0）一页说明

## 平台定位

- 平台性质：专利转让/许可的撮合居间服务平台（不替代法定过户/备案流程）。
- 核心卖点：**订金锁定意向 + 客服跟单 + 证据留痕 + 资金托管**，降低交易不确定性与扯皮成本。
- 数据口径：P0 以“用户上传/后台录入”为准；不宣称“官方实时数据”。

## 客户端范围

- P0：微信小程序（买家/卖家） + 用户 H5（电脑端可用） + PC Web 管理后台（运营/客服/财务）。
  - P0 电脑端：仅浏览+咨询；下单/支付引导回小程序（二维码/链接）。

## 端到端主链路

1) 卖家认证 → 发布上架草稿 → 上传权属材料 → 提交审核 → 上架  
2) 买家搜索/筛选 → 在线咨询 → 支付订金 → 分配客服跟单  
3) 线下签署合同（转让/许可）→ 后台“合同签署确认”写入成交价并解锁尾款  
4) 买家线上支付尾款（平台托管）→ 线下办理权属变更/备案  
5) 后台“变更完成确认”→ 生成结算台账 → 财务人工放款回传凭证（P0）→ 订单完成归档

## 资金与结算（P0 默认参数，可后台配置）

- 订金：一口价按比例 `5%`（`min=¥1,000`，`max=¥20,000`）；面议固定 `¥2,000`；订金计入成交价。
- 自动秒退：支付后 `30` 分钟内且客服未介入线下处理，可无理由原路退订金。
- 佣金：成交价 `5%`（`min=¥1,000`，`max=¥50,000`），**卖家承担**（从卖家应收款中扣除）。
- 尾款：买家线上支付进入平台托管；线下签完合同后再付尾款。
- 放款：必须“变更完成确认”才进入可结算；P0 默认**财务线下人工放款**并上传凭证（不启用超时自动放款）。
- 发票：订单完成后如需开票，财务线下开票并在后台上传电子发票文件（仅对**平台服务费/佣金**开票；不对标的价款开票）。
