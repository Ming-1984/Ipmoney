# 《发票与税务说明》（P0 口径，需财务/法务确认）
> 本说明用于平台前台展示与客服/财务执行口径统一，不构成税务/法律意见。最终以平台主体资质、税务机关要求与实际开票系统规则为准。

## 1. 平台开票范围（非常重要）

- 平台提供的是**居间撮合/交易服务**；平台收取的费用为**平台服务费/佣金**（P0：佣金由卖家承担，从卖家应收款中扣除）。
- **平台仅对“平台服务费/佣金”开具发票**。
- **平台不对专利转让/许可标的价款（成交价）开具发票**：如买家需要标的价款发票，应与卖家线下另行约定并由卖家开具（或按双方合同约定处理）。

## 2. 开票形式与交付（P0）

- P0 默认开具：**电子普通发票**（PDF/图片文件）。
- 交付方式：**平台内下载**（订单详情/发票入口）。**P0 暂不提供邮箱投递**。
- 发票金额：默认等于该订单的 `commission_amount`（服务费金额）。
- 发票项目/服务内容：默认文案为 `居间服务费`（可后台配置）。

## 3. 时机与边界（P0）

- 仅允许在订单状态 `COMPLETED` 后上传/展示发票。
- **P0 在系统内支持“开票申请 + 状态流转”**：卖家/买家在发票中心提交抬头信息，财务审核后线下开票并上传；**P0 仅提供平台内下载，不提供邮箱投递**。

## 4. 后台处理流程（P0：线下开票 + 上传）

1. 卖家/买家在发票中心提交开票申请（抬头、税号等）
2. 财务审核申请（可退回补充信息）
3. 财务线下开票（使用企业现有开票系统/税控系统）
4. 财务在后台选择对应**已完成订单**，上传电子发票文件（PDF/图片），并（可选）填写发票号码/开票日期
5. 系统保存并通知卖家/买家在平台内下载

## 5. 数据与权限（隐私与审计）

- 发票附件属于敏感资料：仅订单参与方可下载；后台仅财务/管理员可管理（按 RBAC 配置）。
- 所有关键操作留痕：上传/替换/下载等行为写入审计日志（who/when/what）。
- 下载建议使用短期签名 URL，并对下载行为写审计日志。
